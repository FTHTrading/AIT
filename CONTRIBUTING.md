# Contributing

## Rules

1. Do not commit secrets or private documents.
2. Do not publish unsupported medical claims.
3. Do not create token-sale logic.
4. Do not expose private KYC, patient data, formulas, or contracts.
5. Every new system object should include:
   - Type definition
   - Example payload
   - Review status
   - Visibility level
   - Compliance disclaimer if needed

## Branch Naming

```text
feat/proof-engine
feat/claim-registry
feat/rwa-packages
feat/x402-metering
fix/security-boundary
docs/protocol-map
```

## Commit Style

```text
feat: add AIT proof manifest engine
feat: add claim classification workflow
feat: add RWA package generator
fix: enforce private IPFS encryption gate
docs: add protocol flow diagrams
```
